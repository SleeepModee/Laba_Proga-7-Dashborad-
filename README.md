ВАРИАНТ 7: Логи LMS
============================================================
Описание: Журналы активности студентов в образовательной платформе. Анализ позволяет оценивать индивидуальную эффективность обучения.

 Поля:
  • ts (int32): время сессии
  • student_id (int16): ID студента
  • score (float32): балл за задание (0–100)
  • time_min (float32): время выполнения, мин
  • attempts (uint8): количество попыток сдачи
  • diff (uint8): уровень сложности задачи (1–5)

Очистка: time_min < 0 или score ∉ [0,100]. Заменить attempts == 0 на 1. Clip time_min по [Q1, Q3].
Пояснение: Отрицательное время и баллы вне шкалы — ошибки лога. Нулевые попытки невозможны → замените на 1. Время выполнения ограничьте межквартильным диапазоном.

Группировка: По student_id. Средний score, мин attempts.
Пояснение: Для каждого студента посчитайте средний балл и минимальное число попыток, потребовавшихся для сдачи. Это покажет усидчивость и уровень усвоения.

Окно: k=20. Moving avg efficiency (score/time_min). np.diff(attempts) → динамика попыток.
Пояснение: Рассчитайте эффективность (балл/время) и усредните её в окне. Изменение количества попыток между задачами покажет прогресс или регресс.

Акцент: Деление на 0 при расчёте эффективности.
Пояснение: Время выполнения может оказаться 0 (мгновенная отправка). Добавьте защиту (например, 1e-8 или условие замены), чтобы избежать ошибки выполнения.
